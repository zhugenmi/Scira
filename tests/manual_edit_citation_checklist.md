# 编辑流程 KB 感知 — 手动验证清单

## 前置
- 启动后端：`python -m src.mcp.server`
- 启动前端：`cd frontend && npm run dev`
- 确认 `data/papers/` 下有至少一个知识库（如 rag/）

## 场景 1：KB 命中
1. 上传一份报告到 `data/outputs/`，在聊天框贴 `<report>...</report>` 初始化编辑
2. 输入"增加5篇参考文献"
3. 预期：助手列出 KB 中 5 篇论文，询问"是否使用"
4. 输入"用这些"
5. 预期：助手输出 EDIT SECTION（参考文献节 + 正文 [n] 角标），工作副本更新

## 场景 2：KB 未命中
1. 同上初始化编辑
2. 输入"加5篇关于量子纠缠相关的参考文献"（确保 KB 无此类）
3. 预期：助手告知未找到，询问是否检索
4. 输入"可以"
5. 预期：SSE 推送 edit_search_start → 检索进度 → 检索完成 → 助手列出新论文
6. 输入"用这些"
7. 预期：同场景 1 第 5 步

## 场景 3：检索失败
1. 同场景 2 第 4 步，但断网或 mock run_workflow 抛异常
2. 预期：助手告知"检索失败"，编辑态保留，可继续其他修改
